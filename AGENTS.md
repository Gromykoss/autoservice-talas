# Автосервис Талас — AGENTS.md

Проект: WhatsApp AI-агент для автосервиса в Таласе, Кыргызстан.
Статус: Архитектура / КП. Разработка не начата.
Дата: 2026-07-16

## Контекст

- **Регион:** Талас, Кыргызстан (~35К население)
- **Клиент:** владелец автосервиса, 1-5 механиков, слабые технические знания
- **Каналы:** WhatsApp (основной), звонки, личное общение
- **Платежи:** наличные, карты
- **Поставщики запчастей:** Бишкек (по телефону/WhatsApp)
- **Текущие процессы:** запись в блокнот/Excel, звонки, ручной учёт

## Принцип

Архитектура как у Алихана (ЕЖО) и ACA (лидогенерация): **агент как операционный слой бизнеса, не чат-бот.**
Клиент платит за токены + разовый setup, без ежемесячных подписок.

## Архитектура

```
Клиент (WhatsApp) → Hermes Bridge :3000 → main.py (poll 3s)
                                              ↓
                                         Router
                    ┌──────────────────────┼──────────────────────┐
                    ↓                      ↓                      ↓
              BOOKING                  PARTS                   CRM
         (запись на сервис)     (поиск запчастей)      (история, база)
                    ↓                      ↓                      ↓
                PostgreSQL ←──────────────────────────────────────┘
```

### Компоненты

| Компонент | Файл | Назначение |
|-----------|------|------------|
| main.py | `main.py` | Главный цикл: приём сообщений, обработка команд |
| book.py | `book.py` | Запись: календарь, напоминания, перенос |
| parts.py | `parts.py` | Запчасти: каталог, запрос поставщикам, сравнение |
| crm.py | `crm.py` | База клиентов, история ремонтов, ТО-напоминания |
| db.py | `db.py` | PostgreSQL: клиенты, машины, записи, запчасти |

### Поток Этапа 1 (MVP)

```
Клиент: «запись на среду»
  → Бот: «На среду свободно: 10:00, 14:00, 16:00. Что удобно?»
  → Клиент: «14:00»
  → Бот: «Марка и номер машины?»
  → Клиент: «Toyota Camry A777AA»
  → Бот: «Записал. Ждём вас в среду 14:00. Напомню за час.» 
  → День X: «Через час запись: Toyota Camry A777AA»
  → После сервиса: «Всё сделали?» → сохраняет в историю
```

### Поток Этапа 2 (Запчасти)

```
Клиент: «нужны колодки на Camry 2019»
  → Бот: ищет в каталоге → «Передние или задние?»
  → Клиент: «передние»
  → Бот: отправляет 3 фото/цены поставщикам в Бишкек через WhatsApp
  → Сравнивает: «TRW 2500 сом (2 дня), Akebono 3200 (завтра), Febest 1800 (4 дня)»
  → Клиент выбирает → Бот заказывает
```

### Поток Этапа 3 (CRM)

```
Бот: «Camry A777AA — пора менять масло (прошло 7000 км). Напомнить клиенту?»
  → Владелец: «да»
  → Бот пишет клиенту: «Ваша Camry проехала 7000 км после замены масла. Записать?»
```

## Стек

| Компонент | Технология | Причина |
|-----------|------------|--------|
| WhatsApp | Hermes Bridge :3000 | Уже работает на Алихане, без Evolution API |
| AI | Hermes Agent + DeepSeek/xAI | BYO keys, pay-per-token |
| БД | PostgreSQL | Бесплатно, надёжно |
| Хостинг | VPS (существующий) | Уже есть |

## База данных

Таблицы:
- `clients` — имя, телефон, язык (ru/kg)
- `cars` — клиент, марка, модель, год, госномер, VIN
- `appointments` — клиент, машина, дата/время, тип работ, статус
- `repairs` — запись, что сделано, запчасти, сумма
- `parts_catalog` — название, совместимость, поставщики, цены

## Правила строительства

1. Никаких подписок для клиента. Разовый setup + токены.
2. WhatsApp как единственный интерфейс. Никаких веб-приложений.
3. Русский + кыргызский язык.
4. Минимальная поддержка: бот работает сам, владелец только подтверждает.
5. Сначала песочница (тестовый WhatsApp), потом реальные клиенты.
6. Пилот на 1 сервис → масштабирование.
